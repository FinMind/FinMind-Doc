# 服務狀態與可用率

FinMind 提供對外即時狀態頁：

[**status.finmindtrade.com**](https://status.finmindtrade.com){ target="_blank" }

該頁顯示 API 目前狀態、過去 **24 小時**與 **90 天**的可用率（uptime），以及歷史事件紀錄。

## 可用率怎麼計算

可用率以**真實 API 流量、逐分鐘**衡量：

- 每一分鐘，若該分鐘內 **server-side error 佔有效請求的比例超過 5%**，該分鐘記為 **down（中斷）**。
- **server-side error** 指 HTTP **5xx** 回應，以及因伺服器端逾時而中斷的連線（HTTP **499**；多半是伺服器忙碌/變慢造成連線被中斷，視為伺服器端問題）。
- 一般 **client error**（例如 `400` / `403` / `404` / `429` 等請求端錯誤）與**沒有流量的分鐘**，**不計入**可用率。

公式：

```text
可用率 % =（有流量的分鐘數 − 中斷的分鐘數）÷ 有流量的分鐘數
```

- **24 小時**與 **90 天**數字由上述逐分鐘結果彙總；90 天與當月數字以流量加權。
- **計畫性維護**時段可排除，不計入中斷。

## 狀態分級

| 狀態 | 含義 |
| --- | --- |
| **Operational** | 該區間沒有中斷分鐘 |
| **Degraded** | 有中斷，但可用率 ≥ 90% |
| **Outage** | 可用率 < 90% |
| **No data** | 該區間沒有流量 |

!!! tip
    狀態頁底部也內含同樣的「How uptime is calculated」說明，可隨時查閱。
